Spring AI provides a unified abstraction for working with vector databases. The core interface is VectorStore, which defines two key operations: add() for storing document embeddings and similaritySearch() for retrieving them. This abstraction supports multiple backends including PgVector, Redis, Chroma, Pinecone, Qdrant, Weaviate and Milvus. The vector store works together with an EmbeddingModel to automatically convert text to vector embeddings before storage and convert queries to vectors during search. When adding documents, you can attach metadata like source filename and chunk index, which will be preserved in search results.
